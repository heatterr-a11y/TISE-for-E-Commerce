Authentication

TISE uses API keys passed in the Authorization header. All requests must be made over HTTPS. We support multiple authentication methods.

Header Authentication 
Authorization: Bearer tise_live_xxxxxxxxxxxxxxxx
X-API-Version: 2024-03-07
X-Client-ID: your-client-id

🔐 Security Best Practices

Never expose your API keys in client-side code. Use environment variables and implement key rotation every 90 days.

Rate Limits

Rate limits are applied per API key. Exceeding limits returns 429 status with Retry-After header.

POST /v1/events/ingest

Stream behavioral events to TISE for real-time analysis. Events processed within 50ms. Supports 40+ event types.

Request Body

Field Type Required Description
user_id string (UUID) Required Unique visitor identifier
session_id string (UUID) Required Current session identifier
events array Required Array of behavioral events (max 100)
timestamp integer (ms) Optional Unix timestamp, defaults to now
metadata object Optional Device, referrer, UTM params
