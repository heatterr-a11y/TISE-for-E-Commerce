Header Format 
Authorization: Bearer tise_live_xxxxxxxxxxxxxxxx

POST /v1/events/ingest

Stream behavioral events to TISE for real-time analysis. Events are processed within 50ms and fed into our temporal prediction model. Supports single or batch ingestion.

Request Body

Field Type Required Description
user_id string Required Unique visitor identifier (UUID v4)
session_id string Required Current session identifier
events array Required Array of behavioral events
metadata object Optional Additional context (device, referrer)

WEBSOCKET /v1/ws/{client_id}

Real-time bidirectional stream for live prediction updates. Connect to receive instant abandonment alerts and send intervention confirmations. 

const ws = new WebSocket(
  'wss://api.tise.io/v1/ws/your-client-id',
  [], 
  { 
    headers: { 
      Authorization: 'Bearer tise_live_xxx',
      X-Client-Version: '1.0.0'
    } 
  }
);

ws.onopen = () => {
  // Subscribe to specific events
  ws.send(JSON.stringify({
    action: 'subscribe',
    channels: ['abandonment_alerts', 'intent_updates']
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  
  switch (data.type) {
    case 'abandonment_alert':
      handleAbandonmentRisk(data);
      break;
    case 'intent_update':
      updateIntentScore(data);
      break;
    case 'intervention_delivered':
      confirmIntervention(data);
      break;
  }
};

Message Types

Type Direction Description
abandonment_alert Server → Client High-risk user detected with prediction data
intent_update Server → Client Periodic intent score updates (every 5s)
intervention_triggered Client → Server Confirm intervention deployment
conversion Client → Server Report successful cart recovery
heartbeat Bidirectional Keep connection alive (30s interval)
