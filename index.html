Quick Start

Get up and running with TISE in under 5 minutes. Install our JavaScript snippet and start streaming behavioral events.

// Initialize TISE tracker
const tise = new TISE({
  apiKey: 'tise_live_xxxxxxxx',
  shopId: 'your-shop.myshopify.com'
});

// Start tracking automatically captures:
// - Mouse movements & velocity
// - Scroll depth & patterns
// - Time on element hover
// - Form field interactions
tise.init();

// Manual event tracking (optional)
tise.track('add_to_cart', {
  product_id: '12345',
  value: 99.99,
  currency: 'USD'
});
POST /v1/events/ingest

Stream behavioral events to TISE for real-time analysis. Events are processed within 50ms and fed into our temporal prediction model. Supports batch ingestion up to 100 events per request.

Request Parameters

Parameter Type Required Description
events array Required Array of behavioral events (max 100)
user_id string Required Unique visitor identifier (UUID)
session_id string Required Current session identifier
timestamp integer Optional Unix timestamp (ms), defaults to now

WEBSOCKET /v1/ws/{client_id}

Real-time bidirectional stream for live prediction updates. Connect to receive instant abandonment alerts and send intervention confirmations. Ideal for real-time dashboard updates and instant trigger systems. 

// Connect to WebSocket stream
const ws = new WebSocket(
  'wss://api.tise.io/v1/ws/your-client-id',
  [], 
  { headers: { Authorization: 'Bearer tise_live_xxx' } }
);

ws.onmessage = function(event) {
  const data = JSON.parse(event.data);
  
  if (data.type === 'abandonment_alert') {
    // Trigger intervention immediately
    showExitIntentModal(data.user_id, data.recommended_offer);
  }
};

// Send intervention confirmation
ws.send(JSON.stringify({
  type: 'intervention_triggered',
  user_id: 'usr_abc123',
  action: 'discount_10_percent',
  timestamp: Date.now()
}));
Message Types
Type Direction Description
abandonment_alert Server → Client High-risk user detected, includes prediction data
intent_update Server → Client Periodic intent score updates (every 5s)
intervention_triggered Client → Server Confirm intervention deployment
conversion Client → Server Report successful cart recovery

Webhook Integration

Configure webhooks to receive real-time HTTP callbacks when abandonment predictions are generated. TISE retries failed deliveries with exponential backoff.

POST /v1/webhooks/subscribe
Subscribe to Webhook Events 
curl -X POST 'https://api.tise.io/v1/webhooks/subscribe' \
  -H 'Authorization: Bearer tise_live_xxxxxxxx' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://your-store.com/webhooks/tise",
    "events": ["abandonment.predicted", "cart.recovered", "intervention.clicked"],
    "secret": "whsec_your_webhook_secret",
    "metadata": {
      "store_id": "store_123",
      "environment": "production"
    }
  }'
